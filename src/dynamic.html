Dynamic semantics tests

Initial State

When given as input:

Question Value 
    form "" {
        "x" x: boolean
        "y" y: integer
        "z" z: string
    }

Should evaluate to:

{x: false, y: 0, z: ""}

Computed Questions Are Initialized

When given as input:

Question Value 
    form "" {
        "x" x: boolean = true
        "y" y: integer = 42
        "z" z: string = "hello world!"
    }

Should evaluate to:

{x: true, y: 42, z: "hello world!"}

Disabled Questions Are Not Changed

When given as input:

Question Value
x 10 
    form "" {
        if (1 > 2) "x" x: integer
    }

Should evaluate to:

{x: 0}

Questions Retain Values Across Visibility Changes

When given as input:

Question Value
x true
y 42
x false
z 2
x true 
    form "" {
        "X" x: boolean
        if (x) 
            "Y" y: integer
        "Z" z: integer
        "Sum" sum: integer = z + y
    }

Should evaluate to:

{x: true, y: 42, z: 2, sum: 44}

Referencing Works Out Of Order

When given as input:

Question Value
x 10 
    form "" {
        "y" y: integer = x
        "x" x: integer
    }

Should evaluate to:

{x: 10, y: 10}

Fixpoint Evaluation

When given as input:

Question Value
c 10 
    form "" {
        "A" a: integer = c+b+1
        "B" b: integer = c+1
        "C" c: integer
    }

Should evaluate to:

{a: 22, b: 11, c: 10}

Multiplication Has Higher Precedence Than Addition

When given as input:

Question Value 
    form "" {
        "X" x: integer = 1 + 2 * 3
    }

Should evaluate to:

{x: 7}

Subtraction Associates To The Left

When given as input:

Question Value 
    form "" {
        "X" x: integer = 1 - 2 - 3
    }

Should evaluate to:

{x: -4}

Parentheses Override Associativity

When given as input:

Question Value 
    form "" {
        "X" x: integer = 1 - (2 - 3)
    }

Should evaluate to:

{x: 2}
